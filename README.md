# 現場の制約を考慮したスーパー向けシフト自動作成モデル 

本プロジェクトは、私が勤めるスーパーのレジ・売場業務における複雑な勤務制約をモデル化し、数理最適化（制約プログラミング）を用いて最適な日間・月間シフトを自動生成するモデルです。
自身がアルバイトとして勤務する中で目にした社員がシフト作成に時間がかかっているという現場の課題を解決するために開発しました。

---

##  導入技術と選定理由

- **Python**
- **OR-Tools (Google CP-SAT Solver)**
  - **選定理由**: シフト作成は「特定の時間にレジに1人必要」「高校生は21時まで」といった厳格な制約が多数絡み合う「制約充足問題（CSP）」です。厳密なルールを高速かつ確実に満たす解を導出できるため、Googleの高性能ソルバーである CP-SAT を採用しました。


---

## ファイル構成とデータフロー

本システムは以下の役割ごとにモジュール化しています。

1. `main.py`: システムの実行エントリポイント。対象年月の設定および全体の統括。
2. `run_monthly_process.py`: 月間（21日〜翌月20日）のループ制御。前日までの「累計労働時間」や「連勤数」を動的に次回の日間ソルバーへ引き継ぐ司令塔。
3. `solve_single_day.py`: 特定日のシフトを30分スロット（計30スロット）単位で人員を配置する数理最適化モデル。
4. `summarize_shifts.py`: ソルバーが吐き出した 0/1 のデータを受け取り、人間が読める「開始時刻 〜 終了時刻」の連続した勤務ブロックに整形するフォーマッタ。
5. `conig.py`: 従業員の希望休やスキルなど自動シフト作成を行う際に基となるデータ（実在する人物とは関係のないダミーデータです)。

---

## シフト自動作成モデルに含まれる制約

`solve_single_day.py` では、例として実運用に耐えうる以下のルールを数式に翻訳しています。

### 【厳格に守るルール（Hard制約）】
- **休憩を入れるための制約**: 勤務時間が6時間を超える場合、1時間の休憩（2スロット）を挿入。
- **属性別時間制限**: 高校生バイトは22:00（スロット27）以降の深夜勤務を禁止。
- **スキルの考慮**: 異なるレジ業務や事務作業など、業務可能者のみを割り当て。

### 【落とし所を探るルール（Soft制約 / ペナルティ関数）】
すべてを絶対ルールにすると解なしになるため、以下の項目は目的関数（Minimize）で点数化し、ソルバーに最適なシフトを計算させています。
- **曜日別の必要レジ台数**: 平日（ピーク時のみ2台）と土日祝（終日2台）の必要レジ数をソフト制約化し、不足時にペナルティ。
- **業務一貫性**: 30分ごとの不必要な「担当業務の移り変わり（レジ→売場への頻繁なスイッチ）」を最小化。
- **土日祝のアルバイト出勤を促進**: 土日祝において、社員・パートが短時間勤務（6時間未満）になってしまう割り当てに強いペナルティを課し、バイトのシフトを優先。

## 課題


当初、従業員、日にち、時間、業務内容の4変数(e,d,t,w)でモデルを組もうとしていましたが、ローカルパソコンの処理能力では耐えられず、２日分生成するのが限界でした。そこで日にちの変数を消去し、従業員、時間、業務内容の3変数(e,t,w)で１日ずつ計算するプログラムにすることで１ヶ月（30日）分を生成することに成功しました。しかし、１日ずつ従業員を割り当てていくので、月全体のバランスを考慮させることができず、月の初めで特定の従業員を集中して投入してしまい、従業員ごとに偏りが出てしまったり、月の途中で主要なパートタイム従業員の月間労働時間の上限に達してしまうなどの問題が発生してしまいました。１日の生成が終了した後に、これまでの合計労働時間を計算して従業員を並び替えたリストをソルバーに渡したり、土日祝はアルバイトを優先的に配置させるようにペナルティを加えたりすることで、より実際のものに近いシフトを自動作成することができるようになりました。

## これから取り組んでいきたいこと

### 月間モデルと日間モデルによる２フェーズの取り組み
１日ごとにモデルを組むことが、ローカルパソコンでの処理限界で最適解だと考えておりましたが、この取り組みを行なっていた春休みが終わり、新学期が始まって時間が経過しClaude Codeというものを知り、導入してより良いアプローチはないかと探っていたところ、１ヶ月の割り振りと１日の割り振りを分けて行うことで、限られた処理能力で１ヶ月のバランスを取ったシフトが自動作成できる可能性が生まれました。これまでは、1日ごとに出勤できる従業員の中から自由に配置していましたが、１ヶ月の労働時間の上限や連続勤務日数を考慮して、その日の従業員を先に配置した後に１日のシフトを組むという２フェーズ構成のモデルを組んでいきたいと考えております。

### 機械学習の取り組み
自動作成した結果から自らの経験に基づいて、実際のシフトに近くなるように修正し、機械学習のライブラリであるscikit-learnを用いて、実際のシフトの傾向を機械学習させ、自動作成に活かそうと試みましたが、学習データが少なかったことや機械学習の知識がまだ乏しく効果的に活用させることができなかったため、断念しました。卒業研究では、このモデルを絡ませながら機械学習を研究してより精度の高いシフトを自動作成させる取り組みをしていきたいと考えております。
